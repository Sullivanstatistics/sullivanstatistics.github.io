Enter Blogdown

There are many ways to build pages on github, the primary one has been to use jekyll. One issue with this is that jekyll only accepts markdown files. In class we have been seeing how amazing it can be to use Rmarkdown documents to include our R code, graphs and analysis.

Blogdown is an R package that creates blog style sites with a new static website called Hugo. On top of that blogdown is a way to incorporate RMarkdown documents into your website. Through this you can actually display your R code and projects. Then like any other blog site you can discuss your favorite things and even code analysis which would allow others to see your coding abilities and interests.

We need to get started with blogdown now:

  1. Install Blogdown:
install.packages("devtools")
devtools::install_github('rstudio/blogdown')
  1. Start a new project.
  • In the right hand corner of your screen start a new project in an empty directory.
  1. Once you have this new project run the following code:
library(blogdown)
install_hugo()
new_site()
  1. Now you can see what your site looks like initially:
serve_site()
  1. Proceed to your next task of setting this project up.

Setting Up Your Page

You now should have a very basic website. The next goal would be to install some themes and actually create some content. In order to do that I have set up an example site that you can see, here. There are quite a few things blank but I wanted to display the capabilities for you. Now here is how you can get a similar page:

  1. Go into git and clone this repository on your computer:

    git clone https://github.com/Sullivanstatistics/GitHubPage.git
  2. Now open up the folder: ```
  • GitHubPage -themes hugo-academic ``Now copy thishugo-academic` folder to the themes folder in your project.
  1. Go back to the GitHubPage main folder. Copy config.toml over to your main page and replace your old config.toml.
  2. Now open up the folder: ```
  • GitHubPage -content ``Copy thehome` folder over to your contents.
  1. You should be able to see the page I created now, if not run serve_site() to see.
  2. Open up config.toml and replace with your information.
  3. Go into the /project/content/home folder and go through all the files in there replacing your information.
