Syllabus

February 23, 2017 in Syllabus

Course Description

Statistical computing is an essential part of analysis. Statisticians need not only be able to run existing computer software but understand how that software functions. Students will learn fundamental concepts – Data Management, Data types, Data cleaning and manipulation, databases, graphics, functions, loops, simulation and Markov Chain Monte Carlo through working with various statistical analysis. Students will learn to write code in an organized fashion with comments. In order to best learn computing students with learn both R and Julia side by side. Each unit in the first half of the course will compare and contrast these two languages to better build an understanding of how to code. This course will use a variety of web-based material from:

  • R
  • Julia
  • RStudio
  • JuliaBox

Course Format

This course will be taught in a ``flipped" format. Students will watch a series of videos and work through some simple coding examples before coming to class. The sequence of these will be displayed on the course website as well as a calendar for which videos need to be watched prior to attending class.

The classroom format will focus on labs and projects. You will be involved with computing and coding on a regular basis. Labs will form into projects which you will finish outside of class and turn in for grading.

Flipped Format and References

We need to take some time to discuss what it means to have a flipped format class. In this style of formatting the note taking portion of the course will be done via videos. Think of these as video textbooks in which you gather the basic details before practicing the material.

In the traditional format this course would have the instructor teaching 160 minutes a week with some hands on lab activities but most coding work done outside of class. In the flipped format you will have access to these lectures as your work prior to class. You then will have 180 minutes of hands on coding projects, code sharing and time with the instructor.

This course will be challenging and will move quickly. It will require on average 10 - 12 hours a week outside of class.

This course will utilize a wealth of materials from many different resources aside from the textbooks:

  • The majority of the R and Julia videos have been created by the instructor and will be linked into the webpages for the course.
  • The R exercises are created by the instructor but have been hosted by DataCamp Lite
  • The website for the course as well as the template come from RStudio RMarkdown Websites
  • Some material is adapted from:

    Shalizi, CR and AC Thomas (2014). Statistical Computing 36-350: Beginning to Advanced Techniques in R. URL: http://www.stat.cmu.edu/~cshalizi/statcomp/14.

  • Also From

Course Objectives

After the successful completion of this course, you will understand and be able to implement the fundamental principles of statistical computing in R, Julia and Python. In particular these include the following capabilities:

  1. Obtain and work with Data.
  2. Clean and Transform data into useable dataframes.
  3. Create Graphics.
  4. Understand the writing and use of functions.
  5. Working with larger dataframes efficiently.
  6. Perform Statistical Optimizations
  7. Code and run an MCMC.
  8. Data visualization.
  9. Relational Databases

Overall Course Expectations

Students in this course will be expected to do the following:

  1. Attend all in class session and actively participate.

  2. Complete all assigned material prior to coming to class and actively participate in class discussions.

  3. Work on weekly projects with classmates.

  4. Complete and turn in all assignments on time. All coding exercises will be submitted via Github.

  5. Respect each other, each others questions and each others discussion.

Evaluation Category Details

Participation

Participation will be calculated by the successful completion of videos and practice coding done prior to class as well as being present and engaged during the in class portion.

With the class meeting once a week it is crucial that all students attend. Any student who needs to miss a class must inform the instructor by 9 am the morning of the class. Failure to do so may result in loss of credit for that class.

Feedback on Students Code

All assignments will be complete in groups. When assignments are turned in you are expected to comment on other students code. This commenting will allow you to see other students coding and logic behind how they solved the problem. Comment to suggest easier solutions, Code that could be removed, and comment on things that do not work or are not clear.

No one works on code alone or creates everything from scratch. If you use another students code or code from the internet please state this. If you notice your code in another assignment comment if this has not been stated.

Weekly Projects

Projects will be a culmination of in class labs with some extra parameters associated with them. Most of the work on Projects will be done in groups.

Final Project

The final project will be a group project. This will either be on data visualization or developing a new package. The final project is expected that you will spend approximately 30 hours on it. This means if completed in a group you will have 30 hours from each member. Projects need to display this level of time involved.

Brown ScM/AM Competencies

Primary Competencies

  1. Identify and implement statistical techniques and models for analysis of data.
  2. Attain proficiency in management, documentation of study data for use in practical statistical analysis.

Refresher Competencies

  2. Apply programming skills to analyze data and develop simulation studies.
  3. Develop proficiency in making oral, written and poster presentations of work to statistical and non-statistical colleagues.
